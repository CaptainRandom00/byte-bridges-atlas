Known caveats
fs.watch on macOS doesn't see editor atomic-saves reliably
Some editors (vim with :w) replace the file atomically, which can confuse fs.watch — change events stop firing.
↳ If serve stops responding to saves, kill + restart. v1.1 candidate: switch to fs.watchFile (polling) or expose a --poll flag.
innerHTML in the bb finance copy
The standalone bb finance generator still uses innerHTML for the sim widget updates — security-hook flagged when copying upstream into this repo.
↳ This repo uses textContent + pre-rendered <b> markers (same render, safer). bb finance dedup sprint will fix theirs.
No JSON Schema document
Schema lives in lib/validate.js + the SKILL.md table. Not a machine-readable JSON Schema (no draft-07 .json). IDE auto-complete works via TS for require/import, not the JSON file.
↳ v1.1 candidate: emit schema.json from validate.js (or vice versa).

BB ATLAS · INTERNALS

What's inside the box

Single-file CLI dispatcher (bin/bb-atlas) routes to one of four lib/ modules. Each module is a single .js file with zero runtime deps. The generator core is split into a pure function (generateString) that returns HTML, plus a file-I/O wrapper (generate) for the common case. Schema validation runs against the same shape the generator consumes — single source of truth.

bb-atlas generate

default

Emit a self-contained interactive HTML atlas. Implicit if first arg is a .json path: `bb-atlas project.json out.html`.

Overview / BB ATLAS · INTERNALS / bb-atlas serve
COMMAND MODULE

bb-atlas serve

live-reload

Watch input.json with fs.watch, regenerate on every save, host at http://localhost:4321 (configurable), live-reload open tabs via SSE.

Debounces fs.watch by 80ms (editors emit multiple events per save). Injects a 5-line EventSource snippet into the served HTML for live-reload. Opens the default browser on boot unless --no-open.
Depends on
lib/serve.jslib/generate.js

Overview / BB ATLAS · INTERNALS / lib/generate.js
LIB MODULE

lib/generate.js

core

Pure-function generator. Exports {generate, generateString}. The same code path serves the CLI, the serve loop, and any programmatic embedding.

generateString(D, dataPathHint?) takes parsed JSON, returns {html, viewCount}. generate({dataPath, outPath}) wraps it with file I/O. Script mode (`node lib/generate.js`) is kept for back-compat with the bb finance copy.
Depends on
no internal dependencies
