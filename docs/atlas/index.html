Known caveats
fs.watch on macOS doesn't see editor atomic-saves reliably
Some editors (vim with :w) replace the file atomically, which can confuse fs.watch — change events stop firing.
↳ If serve stops responding to saves, kill + restart. v1.1 candidate: switch to fs.watchFile (polling) or expose a --poll flag.
innerHTML in the bb finance copy
The standalone bb finance generator still uses innerHTML for the sim widget updates — security-hook flagged when copying upstream into this repo.
↳ This repo uses textContent + pre-rendered <b> markers (same render, safer). bb finance dedup sprint will fix theirs.
No JSON Schema document
Schema lives in lib/validate.js + the SKILL.md table. Not a machine-readable JSON Schema (no draft-07 .json). IDE auto-complete works via TS for require/import, not the JSON file.
↳ v1.1 candidate: emit schema.json from validate.js (or vice versa).

Overview / Internals
BB ATLAS · INTERNALS

What's inside the box

Single-file CLI dispatcher (bin/bb-atlas) routes to one of four lib/ modules. Each module is a single .js file with zero runtime deps. The generator core is split into a pure function (generateString) that returns HTML, plus a file-I/O wrapper (generate) for the common case. Schema validation runs against the same shape the generator consumes — single source of truth.

COMMAND MODULE

bb-atlas generate

default

Emit a self-contained interactive HTML atlas. Implicit if first arg is a .json path: `bb-atlas project.json out.html`.

Resolves output path: explicit 2nd arg → D.output (relative to input dir) → './index.html' beside the input. Creates parent dirs as needed.

Overview / BB ATLAS · INTERNALS / bb-atlas serve
COMMAND MODULE

bb-atlas serve

live-reload

Watch input.json with fs.watch, regenerate on every save, host at http://localhost:4321 (configurable), live-reload open tabs via SSE.

Debounces fs.watch by 80ms (editors emit multiple events per save). Injects a 5-line EventSource snippet into the served HTML for live-reload. Opens the default browser on boot unless --no-open.
Depends on
lib/serve.jslib/generate.js

Overview / BB ATLAS · INTERNALS / lib/generate.js
LIB MODULE

lib/generate.js

core

Pure-function generator. Exports {generate, generateString}. The same code path serves the CLI, the serve loop, and any programmatic embedding.

generateString(D, dataPathHint?) takes parsed JSON, returns {html, viewCount}. generate({dataPath, outPath}) wraps it with file I/O. Script mode (`node lib/generate.js`) is kept for back-compat with the bb finance copy.
Depends on
no internal dependencies

P1 · 1
P1 F-001 Catch-all routing returns 200 for unknown paths OPEN 404 handling
Every /<anything> resolves to the SPA shell with a 200. Latent for the current single-route deployment; blocking once v1.2.0 ships multi-page output.
Fix: Hosting-platform 404 for unmapped paths, or noindex meta on routes outside inventory.json.
▸ Evidence: curl -I /this-does-not-exist → HTTP/1.1 200

P2 F-002 Missing /robots.txt OPEN Robots.txt
No Disallow scope, no Sitemap: reference, no precedent for the next deploy.
Fix: Emit a default User-agent: * / Allow: / / Sitemap: <publicUrl>/sitemap.xml block, or document the deployment-time injection.
lib/generate.js · audit: seo-tech-2026-06-14
P2 F-003 Missing /sitemap.xml OPEN Sitemap
Single indexable URL today, but file's presence is itself a quality signal.
Fix: Add `bb-atlas sitemap <input.json>` subcommand that writes a single-entry sitemap.xml next to the generated HTML when project.publicUrl is set.
audit: seo-tech-2026-06-14
P2 F-004 No canonical tag in generated HTML OPEN Canonical
Preview-URL deployments will compete with production for the same content without an explicit canonical.
Fix: lib/generate.js: emit <link rel=canonical href=`${project.publicUrl}/`> when project.publicUrl is present in the schema.
lib/generate.js · audit: seo-tech-2026-06-14
P2 F-005 noindex not enforced on preview / staging deployments OPEN Meta / OG
Any Vercel/Pages preview URL of the same generated HTML would be indexable too — duplicates production content under preview hosts.
Fix: Emit <meta name=robots content=noindex> when VERCEL_ENV=preview (or equivalent) at generate time.
lib/generate.js · audit: seo-tech-2026-06-14
P2 F-006 OG tags incomplete OPEN Meta / OG
og:title and og:description are partially populated; og:image, og:url, og:site_name are absent. Shareability degraded.
Fix: Schema: add optional project.seo.{title,description,ogImage} overrides. Generator emits the full OG quartet, falling back to project.name + project.tagline when overrides are absent.
lib/generate.js · audit: seo-tech-2026-06-14
